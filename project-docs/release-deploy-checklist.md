# Checklist النشر والمراجعة السريعة

هذه القائمة مخصصة للتنفيذ السريع قبل النشر وبعده، وتعمل جنبًا إلى جنب مع:
- `project-docs/seo-analytics-activation.md`

## قبل النشر

- تأكد من نجاح:
  - `npm run check`
- راجع أحدث الصفحات أو المداخل المضافة.
- تأكد من تحديث:
  - `project-docs/current-status.md`
  - `project-docs/update-log.md`
- إذا كان المطلوب تفعيل العداد، تأكد من وجود:
  - `PUBLIC_GOATCOUNTER_CODE`
  داخل GitHub Variables.

## تنفيذ النشر

- ادفع التعديلات إلى فرع:
  - `master`
- أو شغّل يدويًا:
  - `Deploy to GitHub Pages`
  من تبويب `Actions`.

## بعد النشر مباشرة

- افتح الصفحة الرئيسية:
  - `/`
  - `/ar/`
  - `/en/`
- افتح صفحة مقالة عربية وصفحة مقالة إنجليزية.
- افتح صفحة كتاب عربية وصفحة كتاب إنجليزية.
- تأكد من:
  - سلامة التنسيق
  - عدم وجود صفحة مكسورة
  - ظهور اللغة الصحيحة
  - سلامة التبديل بين العربية والإنجليزية

## مراجعة SEO

- افتح:
  - `/robots.txt`
  - `/sitemap.xml`
- تأكد أن `sitemap.xml` يعمل ويعرض الروابط.
- إذا كانت هذه أول جولة تفعيل أو بعد تعديل مهم:
  - أرسل `sitemap.xml` في Search Console
  - اطلب فهرسة الصفحات الأساسية

## مراجعة العداد

- إذا كان `PUBLIC_GOATCOUNTER_CODE` مضبوطًا:
  - تأكد من ظهور عداد الصفحة الرئيسية
  - تأكد من ظهور عداد الصفحة الحالية في الصفحات الداخلية
- إذا لم يظهر:
  - راجع GitHub Variable
  - أعد النشر

## ما بعد المراجعة

- إذا كان كل شيء سليمًا:
  - دوّن أن النشر ناجح في `update-log` عند الحاجة
- إذا ظهرت مشكلة:
  - سجل موضعها
  - أصلحها في المستودع
  - أعد الفحص ثم أعد النشر
